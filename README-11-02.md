   1. Сделайте `fork` данного репозитория к себе в Github и переименуйте его по названию или номеру занятия, например, https://github.com/имя-вашего-репозитория/git-hw или  https://github.com/имя-вашего-репозитория/7-1-ansible-hw).
   2. Выполните клонирование данного репозитория к себе на ПК с помощью команды `git clone`.
   3. Выполните домашнее задание и заполните у себя локально этот файл README.md:
      - впишите вверху название занятия и вашу фамилию и имя
      - в каждом задании добавьте решение в требуемом виде (текст/код/скриншоты/ссылка)
      - для корректного добавления скриншотов воспользуйтесь [инструкцией "Как вставить скриншот в шаблон с решением](https://github.com/netology-code/sys-pattern-homework/blob/main/screen-instruction.md)
      - при оформлении используйте возможности языка разметки md (коротко об этом можно посмотреть в [инструкции  по MarkDown](https://github.com/netology-code/sys-pattern-homework/blob/main/md-instruction.md))
   4. После завершения работы над домашним заданием сделайте коммит (`git commit -m "comment"`) и отправьте его на Github (`git push origin`);
   5. Для проверки домашнего задания преподавателем в личном кабинете прикрепите и отправьте ссылку на решение в виде md-файла в вашем Github.
   6. Любые вопросы по выполнению заданий спрашивайте в чате учебной группы и/или в разделе “Вопросы по заданию” в личном кабинете.
   
Желаем успехов в выполнении домашнего задания!

### Задание 1

1. `Снижение задержки. Приложение или система извлекают используемые данные из быстрого кеша, вместо того чтобы каждый раз обращаться к более медленному хранилищу. Это значительно ускоряет работу. Как пример, браузер кеширует изображения, CSS и JavaScript файлы. При повторном посещении сайта эти ресурсы загружаются из кеша, что делает загрузку страницы намного быстрее.` 
2. `Уменьшение нагрузки на основную систему. Снижается нагрузка на серверы или базы данных, поскольку им приходится обрабатывать меньше запросов. Как пример, серверы баз данных могут кешировать результаты часто выполняемых запросов. Когда тот же запрос приходит снова, сервер может получить ответ из кеша, не выполняя полный запрос к базе данных.`
3. `Ускорение сложных вычислений. Результаты ресурсоёмких вычислений могут быть сохранены в кеше. Если те же данные встречаются снова, ранее вычисленный результат может быть немедленно возвращен. Как пример, в программном обеспечении для анализа данных, кеширование может сохранять промежуточные или конечные результаты вычислений, чтобы избежать их повторного выполнения.`
4. `Экономия ресурсов. Меньшее обращений к основным ресурсам (сетевые запросы, дисковые операции) означает экономию трафика, процессорного времени и энергии. Как пример, мобильные приложения могут кешировать данные пользователя на устройстве, чтобы уменьшить потребление мобильного трафика при работе в автономном режиме или в условиях плохого соединения.`
5. ` Обеспечение доступности данных. Кеш может служить временным хранилищем данных, даже если основной источник данных недоступен. Как пример, кешируется контент (видео, изображения) на серверах, расположенных ближе к конечным пользователям. Это позволяет пользователям получать доступ к контенту даже при временных проблемах с основным сервером.`
