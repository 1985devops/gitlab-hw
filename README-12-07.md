### Задание 1.

1. `Один мастер и один запасной slave-сервер. Когда есть главный сервер master и один slave-сервер, который готов взять на себя работу, если главный сломается.`

* Всегда доступен, если главный сервер упал, второй быстро его заменит.
* Бэкапы не тормозят если делать на запасном сервере, чтобы основной не отвлекался.
* Данные в безопасности, даже если что-то с основным сервером случится, есть резервная копия.
* Приложение не нужно сильно менять, все запросы идут на один работающий сервер.

2. `Один мастер и много помощников slave-серверов`

* Запросы на чтение распределяются между кучей серверов, что позволяет обслуживать очень много пользователей одновременно.
* Основной сервер не перегружен, он только занимается изменениями, что делает систему стабильной.
* Высокая надежность. Если один из slave-серверов откажет, система продолжит работать, другие slave-сервера подхватят.
* Быстрый доступ по всему миру. slave-сервера можно разместить в разных дата-центрах,в разных странах, чтобы пользователям было быстрее получать данные.


### Задание 2.

1. `Структура таблиц`

 `Структура таблиц пользователи`

* id — уникальный номер каждого пользователя 
* имя — имя пользователя
* email — электронная почта
* город — город проживания

 `Структура таблиц Книги`

* id — уникальный номер каждой книги
* название — название книги
* автор — автор книги
* жанр — жанр
* описание — краткое содержание.

 `Структура таблиц Магазины`

* id — уникальный номер магазина
* название_магазина — название
* адрес — адрес магазина
* баланс — сколько денег на счету магазина


2. `Вертикальное шардирование (разделение по таблицам)`

* Сервер А (Профили) — хранит только таблицу пользователи.
* Сервер Б (Каталог) — хранит только таблицу книги.
* Сервер В (Коммерция) — хранит только таблицу магазины.


3. `Горизонтальное шардирование (шардинг строк). Берём большую таблицу и делим её содержимое (строки) между несколькими серверами. `

 `Таблица «Пользователи`
* Шард 1 — все пользователи с нечётными id
* Шард 2 — все пользователи с чётными id

 `Таблица «Книги`

* Шард 1 — книги, названия которых начинаются на буквы от А до М.
* Шард 2 — книги, названия которых начинаются на буквы от Н до Я.

 `Таблица «Магазины`

* Шард 1 — магазины, расположенные в Москве.
* Шард 2 — магазины, расположенные в Санкт‑Петербурге.

4. `Блок‑схема архитектуры`

* Клиент / Приложение — когда пользователь заходит на сайт или в приложение
* Маршрутизатор запросов — «диспетчер» решает, куда отправить запрос.
* БД Пользователи,БД Книги,БД Магазины — три блока, разделённых вертикально (каждый за свою область).
* Шард 1 и Шард 2 внутри каждого блока — это горизонтальное деление.

5. `Режимы работы серверов`

* Master Основной сервер, кто принимает команды на изменение данных. Добавить нового пользователя, обновить баланс магазина, удалить книгу.
* Replica Сервер‑копии. следит за Master и копирует все изменения. Отвечает только на запросы чтение: показать профиль пользователя, найти книгу, посмотреть адрес магазина. Разгружает основной сервер, беря на себя часть работы.
