### Задание 1. Описание таблиц и типов данных PostgreSQL.

1. `employees (Сотрудники). Основная таблица с кадровыми данными.`

* id (SERIAL) — уникальный внутренний идентификатор сотрудника (первичный ключ).
* full_name (VARCHAR(255)) — Фамилия, имя и отчество.
* salary (NUMERIC(12, 2)) — Ежемесячный оклад с точностью до двух знаков.
* hire_date (DATE) — дата приёма на работу.
* position_id (INTEGER) — ссылка на должность сотрудника (внешний ключ).
* department_id (INTEGER) — ссылка на подразделение сотрудника (внешний ключ).

2. `positions (Должности) .Справочник штатных должностей в компании.`

* id (SERIAL) —  уникальный идентификатор должности (первичный ключ).
* title (VARCHAR(150)) — наименование должности, например «Ведущий разработчик».

3. `departments (Подразделения).Таблица, описывающая структуру компании.`

* id (SERIAL) — уникальный идентификатор подразделения (первичный ключ).
* name (VARCHAR(150)) — название подразделения (например, «Команда разработки API»).
* type_id (INTEGER) — ссылка на тип подразделения (внешний ключ).

4. `department_types(Типы подразделений). Классификатор для иерархии структуры.`

* id (SERIAL) — уникальный идентификатор типа (первичный ключ).
* name (VARCHAR(50)) — категория подразделения: «Департамент», «Отдел», «Группа».

5. `branches (Филиалы). Список географических локаций компании.`

* id (SERIAL) — уникальный идентификатор филиала (первичный ключ)..
* address (TEXT) — полный почтовый адрес офиса.

6. `projects (Проекты). Реестр текущих и завершённых проектов.`

* id (SERIAL) — уникальный идентификатор проекта (первичный ключ).
* name (VARCHAR(255)) — Название проекта.

7. `employee_projects (Назначения на проекты). Связующая таблица для отношения «многие-ко-многим» между сотрудниками и проектами.`

* employee_id (INTEGER) — ссылка на сотрудника (внешний ключ).
* project_id (INTEGER) — ссылка на проект (внешний ключ).
* PRIMARY KEY (employee_id, project_id) —  составной первичный ключ, гарантирующий уникальность назначения одного сотрудника на конкретный проект.

### Задание 2. Определение связей между таблицами

* department_types ↔ departments: Один тип подразделения (например, Отдел) может быть у множества подразделений. (1:N)
* positions ↔ employees: Одна должность (например, Инженер) может быть присвоена нескольким сотрудникам. (1:N)
* departments ↔ employees: В одном подразделении работает множество сотрудников. (1:N)
* branches ↔ departments: На одном филиале может размещаться несколько подразделений. (1:N)
* employees ↔ employee_projects: Один сотрудник может числиться во многих записях о назначении на проекты. (1:N)
* projects ↔ employee_projects: Один проект может иметь множество назначений сотрудников. (1:N) – Связи employees и projects вместе реализуют отношение "многие ко многим".
